# 커밋 메시지 컨벤션 (Commit Message Convention)

팀원 간의 원활한 협업과 변경 이력 추적을 위해 아래의 커밋 규칙을 준수합니다.

## 1. 커밋 메시지 구조 (Structure)

```text
TYPE: [REQ-ID] Subject
```

*   **TYPE**: 변경 사항의 종류 (아래 표 참조)
*   **REQ-ID**: 관련 제품 요구사항 ID (예: REQ-001, REQ-101). 단순 문서 작업이나 리팩토링 등 요구사항과 직접 관련이 없는 경우 `[Common]` 또는 `[Refactor]` 등으로 표기 가능.
*   **Subject**: 변경 내용 요약 (한글 권장, 명령문 사용)

**예시**:
*   `FEAT: [REQ-001] 회원가입 API 구현`
*   `FIX: [REQ-201] 입찰 금액 검증 로직 오류 수정`
*   `DOCS: [Convention] 커밋 메시지 규칙 문서 추가`

## 2. 커밋 타입 (Commit Type)

| **커밋 타입**    | **정의**                        | **예시**                       |
| ------------ | ----------------------------- | ---------------------------- |
| **FEAT**     | 새로운 기능 추가                     | `FEAT: [REQ-101] 상품 등록 API 구현`    |
| **FIX**      | 버그 수정 및 오류 해결                 | `FIX: [REQ-401] 결제 금액 계산 오류 수정` |
| **CHORE**    | 코드 수정 없는 부수적인 변경 (빌드, 의존성 등)  | `CHORE: [Common] 라이브러리 버전 업데이트`       |
| **REFACTOR** | 기능 변화 없이 코드 구조 개선 (리팩토링)      | `REFACTOR: [REQ-202] 입찰 유효성 검사 모듈 분리`    |
| **DOCS**     | 문서 추가 또는 수정 (README, 주석 등)    | `DOCS: [Common] 실행 가이드 업데이트`         |
| **TEST**     | 테스트 코드 추가 또는 수정               | `TEST: [REQ-008] 회원 탈퇴 로직 테스트 추가`  |
| **STYLE**    | 코드 로직에 영향 없는 스타일 변경 (포매팅, 공백) | `STYLE: [Common] 들여쓰기 수정`          |
| **BUILD**    | 빌드 설정 파일 또는 외부 종속성 변경         | `BUILD: [Common] Webpack 설정 수정`    |
| **CI**       | CI/CD 설정 파일 변경                | `CI: [Common] 배포 스크립트 수정`          |
| **PERF**     | 성능 개선 관련 작업                   | `PERF: [REQ-103] 상품 목록 조회 속도 최적화`  |
